Working with Tableau - Solving your Data sets easily

June 2020   |   by Black Board Learning   |   Data Analytics
“The greatest value of a picture is when it forces us to notice what we never expected to see” – John W.Tukey

Tableau is one of the most popular and powerful BI tool for data visualization with a simple drag and drop user interface. It has enormous features and powerful calculations to facilitate vast data exploration. In order to use this enriched software to make the best out of your data, you first need to learn and understand where to start and how to approach your data. In this blog, we will show you a step by step approach to solve your data using the Tableau Desktop software.

We will be using a global data set called Sample Superstore, which comes with every installation of the Tableau software, to demonstrate the steps in solving a data set. You can find the data set Excel file in the following folder inside your installation path: My Tableau Repository\Datasources\2020.2\en_GB-EU\. When you open your Tableau Desktop, you will find options to connect to different data sources ranging from simple spreadsheets to cloud systems. Select Excel Worksheet from the options and then browse to the above mentioned path and open the Sample superstore file.

The Superstore data set contains data about the orders that have been placed from different regions in various product categories.

Once you have opened the file in Tableau, select the “Order” sheet from the left pane. You will be able to see the preview of the data loaded from the data source in the table on the right. You will find a tab below named Sheet1 in the bottom, which will prompt you to Go to worksheet. Go to that worksheet and this where you will be visualizing your data using charts and graphs. In the left Data pane, the variables from your data source will be listed as dimensions and measures depending upon the type of value that particular variable holds. Dimensions hold qualitative data, whereas Measures hold quantitative numerical data.

In this Superstore data set, the qualitative variables such as Product Name, Category, Region, Segment, etc., will come under Dimensions and the numerical variables such as Discount, Profit, Quantity and Sales will come under Measures. You can select, drag, and drop these variables to the columns and rows fields on the top, in order to generate the graphs which will help you understand the patterns in the data. It is always advisable to choose two to four or five variables at a time to have a better and easy understanding.

As a first step, let us get a general understanding of creating a visualization. As you can observe, the orders are from one of the four regions in the United States – Central, East, West, and South. Say, we have to figure out which region has the highest amount of sales overall. For this, we just need two variables – Region and Sales. Now, drag and drop the Region variable in the Columns field and the Sales variable in the Rows filed. A bar chart will be created showing the aggregate sales of each region, from which you will be able to figure out in which region the sales is the highest. You can always change the type of chart from the Show Me option in the top right corner.

In good information visualization, there are no rules, no guidelines, no templates, no standard technologies, no stylebook … You must simply do whatever it takes.

Edward Tufte

Now that you have got an idea of how to create a basic visualization, let us try to dive deeper into our data set. First of all, for any business, Profit will be the most important concern. So here, our area of interest could be which product sub-category is least profitable or in which country the profit is least or why a particular product is low in giving profit and so on. Let us figure these out one by one. Tableau also provides features for us to calculate aggregate measures as a new calculated field. For example, in order to calculate the Profit percentage, right click on any measures filed, click on Create – Calculated field, and create the field by entering the formula[(Sum(Profit)/Sum(Sale))*100] in the dialog box that appears. Now you can use this new field - Profit percentage also for further visualizations.

You can also compare more than two variables in a single visualization by using the Marks available such as Colour, Size, Label, etc., to differentiate each variable. Say, we need to find out the state in which the profit percentage is the lowest in the Central region. Apply the filter in the Region by dragging and dropping the Region in the Filters field and then choosing Central from the options. Now drop Sales in the Rows field so that we can as well compare it with Profit percentage. Drop the State on the colour Marks and Profit percentage on the size respectively. You will be able to get a visualization similar to the following visualization picture.

It can be observed that the State Illinois has the least Profit percentage of -15.73% and the State Minnesota has the highest profit percentage of 36.24% in the Central Region. Likewise you can infer many such information from a single visualization. You can also create different visualizations using the same variables but different combinations or using different variables as well.

You can use different sheets for different visualizations by adding sheets from the tab below, so that you can save all the graphs and charts for future reference also. You can also create new dashboards to have a combined view of different visualizations. You can as well create Stories where you can add narratives and hints to each chart for future reference.

Try rectifying the root cause of any problem instead of providing temporary solutions.

There are many possible inferences that can be made from a given data set. But as an analyst, our focus has to be on finding the main cause of our problem. In the sample superstore data set, we have seen that many products are facing losses. So in order to find out the solution for this, we need to analyse the relationship between Profit and other measures. Profit percentage is an aggregated measure of profit ans sales. So we have to compare the Profit percentage with the remaining measure, which is the discount. Drop Discount in the Columns field and profit percentage in the Rows field.

From the visualization, we can observe that for a discount of 40% and above all orders are unprofitable. If you want to add some more variables for detailed information on the losses you can add it using the different Marks features. For example, adding the sub-category field, could show that the sub-categories such as furnishings and binders always have faced losses since they are given a discount of 60% and above.

The more you explore, the more fun you will have.

Tableau has a wide range of options to describe the data in different views, filtering, formatting, generating trend lines, grouping and clustering, prescribing and forecasting, etc. Start exploring by yourself these vast features and make use of the most suitable options whenever and wherever applicable. The fun of playing with charts and graphs is unlimited in Tableau.

If you are not able to infer any pattern, try changing your approach.

Different data sets may require different ways of approaching and infering. The standard ways always doesn't workout. If you are getting stuck at a particular point or if you feel that your inference might not be valid, do not hesitate to rephrase your entire approach in order to find valuable information.

Ask questions to yourself at every point.

While you are analysing, if you are finding out any descriptive information, try to find out why and how those results are coming, and repeat this process again and again until you find out the root cause of the problem. The more questions you ask, the more accurate your insights will get.
